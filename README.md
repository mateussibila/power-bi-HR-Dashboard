## ğŸ“Š Data

The project is based on HR datasets containing information about employees, including:

- Employee demographics

- Employment status and organizational structure  

These datasets allow the analysis of workforce composition, hiring patterns, retention levels, and turnover behavior.

---

## ğŸ”„ Data Preparation & Modeling

Data preparation and modeling were performed using **Power Query**.

The ETL process included data inspection, column standardization, data type adjustments, handling missing or inconsistent values, and creation of calculated fields for time-based analysis.

An **analytical data model** was then built by creating referenced queries, merging tables, and applying **dimensional modeling** principles, defining fact and dimension tables, establishing relationships, and validating cardinality.

This process ensured a clean, consistent, and well-structured semantic layer for analysis.

---

## ğŸ“ˆ Dashboard Structure

The dashboard is organized into three main analytical sections:

### 1. Headcount
This section provides a high-level overview of the workforce, including total headcount, active employees, and workforce evolution over time. It allows users to quickly assess growth or decline across departments and roles.

### 2. Retention
This section focuses on employee retention metrics, highlighting retention rates by department, job level, and demographic attributes. It helps identify how effectively the organization is keeping its talent.

### 3. Turnover
This section analyzes employee attrition, presenting turnover rates, exit trends, and termination patterns over time. It enables HR teams to detect critical areas with higher employee loss and potential organizational risks.
